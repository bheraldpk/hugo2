    Creating an App #

    https://commure.atlassian.net/browse/PRODUCT-94

    1. Login to the Application Management Tool.
    2. Click the + CREATE button in the top-right corner of the screen.
    3. Complete the following details in the drawer that opens on the right side of the screen and then click Save.
      • Name
      • Display Name
      • Type
      • Path

    Now you have a the shell of your application with a default set of database tables. You can make a number of updates to an app after it has been created, such as defining its database schema, setting business rules, editing basic info, and adding and removing dependancies.

    View app database schema #

    1. Select the app you want to edit in the application list.
    2. Click the Data Schema button. From here you can:
      • Create a new table
      • Create new table fields or edit existing fields
      • Edit existing fields
      • Preview app

    Preview app #

    1. Open the database schema for the app (see View an app’s database schema).
    2. Click the PREVIEW button to open a preview of the app.

    Create a new table #

    1. Open the database schema for the app (see View an app’s database schema).
    2. Click the Create button in the top-right corner of the screen.
    3. Complete the following details in the drawer that opens on the right side of the screen and then click Save.
      • Select Table or Canonical
      • Enter a Name for this table. For canonical tables, this will be a drop-down with the list of canonical tables available. For non-canonical tables, this will be a text input.
      • Enter a Description for this table

    Create or edit table fields #

    For exitsing tables in your app’s database schema, you can add and edit fields.

    1. Open the database schema for the app (see View an app’s database schema).
    2. Select the table in the list that you want to edit.
    3. In the bottom-half of the screen, select the Fields tab and then select the field you want to edit or click the + CREATE button to create a new field.
    4. Complete the following details and then click the Save button.
      • Select the field Type
        • Field - options include: base64Binary, boolean, date, datetime, float, integer, json, money, numeric, serial, string, text, time, uuid.
        • Reference - A reference field stores a reference to a field on another table. For non-canonical tables the field type must be uuid. For canonical tables (FHIR) it must be string.
        • Computed - To add the calculation logic, the user can write a javascript.
        • Choice -
      • Enter the field Name
      • Select the Field Type

    Edit table information #

    1. Open the database schema for the app (see View an app’s database schema).
    2. Select the table in the list that you want to edit.
    3. Click the EDIT button in the top-right corner of the screen to launch the drawer to edit the table information.
    4. Edit the table Name, Is Canonical, or Description, and then click SAVE.
