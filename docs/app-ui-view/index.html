    Overview of App UI Screen #

    The App UI screen is where you can create, edit, and delete UI pages.

    What is a page?

    A page is a unique display used for different purposes, such as changing the view depending on the device (web, mobile, tablet) or creating different views for different audiences, such as managers and staff.

    A page is made up components and is managed by rules.

    Technically, a page is a component that gets mounted when a subpath matches the Page subpath configuration at the Appâ€™s Page Routing Engine.

    An app can have multiple pages, but one page can only be part of one app (as it might depend on its context). A page can render multiple Components, and one Component can be rendered by multiple Pages.

    What are components?

    Components are presentational or data-wired UI Components/elements/functions. Components render design elements on the page to create the overall functuinality and display. A Page typically made up of an assembly of components.

    Viewing the Page List for an App #

    1. Select Applications from the sidebar and then select the app you want to edit.
    2. Click the APP UI button near the top of the screen. This opens the Page List showing the Page Name, Path, and Page icon for each page that has been created for the app. From the page, you can Preview the app, Create a new page, and Edit a page in the app.

    Viewing the Compoments on an App Page #

    1. Open the App UI page for the app you want to edit (for details, see Viewing Page List for an App.
    2. Select the Page you want to view.
    3. Select the Components tab and then click the componenent you want to view.
      The Component detail page displays, including all of the properties you can control. Editable properties are displayed with an Edit icon next to the Property Name.
